# A100 PROJECT CONSTITUTION

## Fixed governance
- The user controls roadmap changes.
- Mid-sprint comments, defects, and ideas do not replace the approved roadmap.
- Current work is classified as Critical, Hotfix, Optimization, Current Sprint, or Future Queue.
- Hotfixes and optimizations are absorbed without turning them into a new product direction.
- Stability, regression prevention, and completion come before unplanned feature expansion.
- Meeting decisions and their reasons are project assets and must remain in the repository.

## Fixed architecture
- Runtime First: live runtime is authoritative.
- Strict Read Only for monitoring and trust queries.
- Snapshot is evidence/recovery, not an alternate runtime authority.
- Certification is evidence-based; synthetic completion is OFF.
- Registry remains 341/341.
- Schema 1, Paper 20, Shadow 60, Live Trading OFF until certified.
- Existing data and configuration must be preserved.
- Railway terminology and workflow are authoritative.

## Product direction
- Independent Long AI and Short AI.
- LONG / SHORT / WAIT consensus.
- Explainable evidence from chart, news, funding, OI, volume, whale, regime, and learning.
- Quiet Monitoring + Daily Brief + high-quality Opportunity Alert.
- Alerts should be rare enough that an A100 alert is worth checking.


## Mobile-first release rule
- Default distribution is a MOBILE FLAT incremental patch.
- No manual folder creation is required for GitHub upload from a phone.
- Full/nested packages are exceptional and require explicit user approval.
