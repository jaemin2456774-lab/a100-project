# LESSONS LEARNED

- Losing meeting context across chats causes repeated discussion and delays stabilization.
- Project memory must ship with the code and be committed to GitHub.
- Do not present already-approved decisions as new proposals.
- Do not change the roadmap because one command exposes a defect.
- Measure first, patch within the sprint, regression-test, then continue the locked roadmap.
- Never claim PASS from registration alone; use real engine, output, store, and replay evidence.
